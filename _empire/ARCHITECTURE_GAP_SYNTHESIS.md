## WHAT “THE EMPIRE” TELLS US ABOUT SKILL ARCHITECTURE

### Pattern 1: Modular Vertical Stacks

Every system in the empire has the same pattern:

```
[Strategic layer] → [Operational layer] → [Execution layer] → [Output layer]
```

The current skill ecosystem has strong Strategic + Operational layers.
Execution layer: partial. Output layer: weak.

**Missing**: skills that take analysis output → push it somewhere real.

### Pattern 2: The Ouro Command Pattern

Ouro was designed as a “plug-and-play” agent with slash commands.
The current skill ecosystem has no command interface layer.
Users type natural language; skills activate on keyword match.
Ouro had explicit, composable commands.

**Missing**: `intake-triage` that replicates the /ouro command pattern.

### Pattern 3: Financial + Legal Always Together

Every version of the empire (Ouroboros Foundation, Meta Umbrella, Aurora) built
financial structure and legal compliance as a core layer — not as afterthoughts.
The current skill ecosystem has 0 financial skills and 0 legal skills installed.

**This is the most historically consistent gap across every system.**

### Pattern 4: The Self-Improvement Loop

/ouro rise + /ouro audit → continuous system improvement
The current ecosystem has agent-eval-harness but no self-evaluation loop.
Nothing compares today’s output to yesterday’s. Nothing triggers improvement.

**Missing**: `outcome-tracker` + `empire-self-audit` skill.

### Pattern 5: Jurisdiction Specificity

The empire was always specifically Australian:

- ATO alignment, Xero (AU), Legislation.gov.au, AustLII
- FVIO/PSIO (Victorian law), DFFH, Magistrates’ Court
- R&D Tax Incentive (AU), Singapore subsidiary, Delaware LLC

The current skill ecosystem has:

- 0 AU-specific skills (the gap analysis identified `legal-document-au` as P0)
- 0 ATO/tax skills
- 0 AU government agency interaction skills (Centrelink, DFFH, ATO, etc.)
